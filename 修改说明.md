# Typecho 导航主题修改说明

## 修改内容

本次修改解决了首页只显示 5 个网址的问题，现在可以显示所有文章。

### 修改的文件

1. **index.php** (第 34 行)
   - 原代码：`'type=category'`
   - 修改为：`'type=category&pageSize=10000'`
   - 作用：获取顶级分类下的所有文章，不受系统默认每页文章数限制

2. **functions.php** (第 73 行)
   - 原代码：`'type=category'`
   - 修改为：`'type=category&pageSize=10000'`
   - 作用：AJAX 请求子分类时也显示所有文章

## 问题原因

Typecho 的 `Widget_Archive` 默认会使用系统设置中的"每页文章数"参数（通常是 5 或 10），导致首页只显示有限数量的卡片。通过添加 `pageSize=10000` 参数，可以强制显示所有文章。

## 使用方法

1. 将修改后的主题文件上传到 Typecho 的 `usr/themes/` 目录
2. 在后台启用该主题
3. 刷新首页，即可看到所有文章以卡片形式展示

## 注意事项

- 如果文章数量超过 10000 篇，可以将 `pageSize` 参数调整为更大的值
- 建议合理控制文章数量，避免页面加载过慢
- 已测试支持响应式布局，在不同屏幕尺寸下自动调整卡片排列

## 技术支持

如有问题，请检查：
1. Typecho 版本是否为 1.2.x
2. 主题文件是否完整上传
3. 浏览器控制台是否有 JavaScript 错误
